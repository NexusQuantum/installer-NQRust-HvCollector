# HV Collector Installer

Interactive TUI installer for deploying the HV Collector stack using Docker Compose.

## Overview

This installer provides a guided setup for the HV Collector platform, which includes:
- **PostgreSQL** - Time-series metrics database (port 5431)
- **Hypervisor Collector** - Kubernetes metrics collection service
- **Prometheus Port-Forward** - Metrics forwarding via kubectl
- **Fluentd** - Log aggregation to PostgreSQL

## Prerequisites

1. **Docker** + **Docker Compose v2** - [Install Docker](https://docs.docker.com/get-docker/)
2. **Docker Buildx** (BuildKit) - Usually included with Docker CE
3. **Rust** (for building from source) - [Install Rust](https://rustup.rs/)
4. **GitHub Personal Access Token** with `read:packages` scope
   - Required for pulling GHCR images
   - [Create PAT](https://github.com/settings/tokens/new)
5. **Kubeconfig file** - Valid `kubeconfig.yaml` for your Kubernetes cluster

## Quick Start

### Option A: Install from Release
```bash
# Download latest .deb
curl -LO https://github.com/NexusQuantum/installer-NQRust-HvCollector/releases/latest/download/nqrust-hvcollector_*.deb

# Install
sudo dpkg -i nqrust-hvcollector_*.deb

# Run installer
nqrust-hvcollector install
```

### Option B: Build from Source
```bash
git clone https://github.com/NexusQuantum/installer-NQRust-HvCollector.git
cd installer-NQRust-HvCollector
cargo build --release
./target/release/nqrust-hvcollector install
```

## Installation Steps

1. **Prepare** `kubeconfig.yaml` in project directory
2. **Run** `nqrust-hvcollector install`
3. **Authenticate** with GitHub PAT when prompted
4. **Configure** PostgreSQL and Hypervisor credentials:
   - PostgreSQL: host, user, password, port (5431), database, schema
   - Hypervisor: host, user, password
   - Collector: cluster name, interval (default: 60s)
5. **Proceed** - Installer builds images and starts services

## Post-Installation

```bash
# Check service status
docker compose -p hvcollector ps

# View collector logs
docker compose -p hvcollector logs -f hypervisor-collector

# View Fluentd logs
docker compose -p hvcollector logs -f fluentd

# Connect to PostgreSQL
docker exec -it postgres psql -U postgres -d hypervisor
```

## Configuration Files

- `.env` - Environment variables (generated by installer)
- `kubeconfig.yaml` - Kubernetes cluster credentials (user-provided)
- `docker-compose.yaml` - Service definitions (embedded)
- `fluentd/fluent.conf` - Fluentd configuration (auto-generated)

## Airgapped Installation

For environments without internet access:

1. **Build airgapped binary** (on internet-connected machine):
```bash
cargo build --release --features airgapped
```

2. **Transfer binary** to airgapped environment
3. **Run**: `./nqrust-hvcollector install`

All required Docker images are embedded in the airgapped binary.

## Troubleshooting

### Services Won't Start
```bash
# Check logs
docker compose -p hvcollector logs

# Verify .env file
cat .env

# Check kubeconfig
kubectl --kubeconfig=./kubeconfig.yaml get nodes
```

### PostgreSQL Connection Errors
- Verify PostgreSQL host is reachable
- Check port 5431 is not in use: `ss -tulpn | grep 5431`
- Confirm credentials in `.env`

### Collector Not Collecting
- Check kubeconfig is valid
- Verify Prometheus URL (if provided)
- Check collector logs: `docker compose logs hypervisor-collector`
