# HV Collector Installer

Interactive TUI installer for deploying the HV Collector stack using Docker Compose.

## Overview

This installer provides a guided setup for the HV Collector platform, which includes:
- **PostgreSQL** - Time-series metrics database
- **Hypervisor Collector** - Kubernetes/Harvester metrics collection service
- **Prometheus Port-Forward** - Metrics forwarding via kubectl
- **Fluentd** - Log aggregation to PostgreSQL

## Prerequisites

1. **Docker** + **Docker Compose v2** - [Install Docker](https://docs.docker.com/get-docker/)
2. **Docker Buildx** (BuildKit) - Usually included with Docker CE
3. **Kubeconfig file** - Valid `kubeconfig.yaml` for your Harvester/Kubernetes cluster

## Quick Start

### Option A: Airgapped Binary (Recommended)

Download the latest airgapped binary from the [Releases](https://github.com/NexusQuantum/installer-NQRust-HvCollector/releases) page.

```bash
# Verify checksum
sha256sum -c nqrust-hvcollector-airgapped.sha256

# Make executable
chmod +x nqrust-hvcollector-airgapped

# Place kubeconfig.yaml in the same directory, then run:
./nqrust-hvcollector-airgapped
```

The airgapped binary includes all Docker images â€” no internet access required during installation.

### Option B: Build from Source

```bash
git clone https://github.com/NexusQuantum/installer-NQRust-HvCollector.git
cd installer-NQRust-HvCollector
cargo build --release
./target/release/nqrust-hvcollector
```

## Installation Steps

1. **Prepare** `kubeconfig.yaml` in the same directory as the installer binary
2. **Run** `./nqrust-hvcollector-airgapped`
3. **Configure** via TUI form:
   - PostgreSQL: host (`postgres`), user (`postgres`), password, port, database (`hypervisor`), schema (`fluentd`)
   - Hypervisor/SSH: host, user, password
   - Collector: interval (default: 60s), Prometheus URL (optional), retention days
4. **Proceed** â€” Installer starts all services via `docker compose up -d`

## Post-Installation

```bash
# Check service status
docker compose -p hvcollector ps

# View collector logs
docker compose -p hvcollector logs -f hypervisor-collector

# View Fluentd logs
docker compose -p hvcollector logs -f fluentd

# Connect to PostgreSQL
docker exec -it hypervisor-postgres psql -U postgres -d hypervisor
```

## Services

| Service | Container | Description |
|---|---|---|
| postgres | hypervisor-postgres | PostgreSQL 15 database |
| hypervisor-collector | hypervisor-collector | Harvester/K8s metrics collector |
| prometheus-pf | hypervisor-prometheus-pf | Prometheus port-forward (kubectl) |
| fluentd | hypervisor-fluentd | Log aggregation (ports 24224, 9880) |

- `docker-compose.yaml` - Service definitions (embedded in installer)

## Troubleshooting

### Services Won't Start
```bash
# Check logs
docker compose -p hvcollector logs

# Verify .env file
cat .env

# Check kubeconfig
kubectl --kubeconfig=./kubeconfig.yaml get nodes
```

### PostgreSQL Connection Errors
- Verify PostgreSQL password in `.env`
- Check port is not in use: `ss -tulpn | grep <port>`
- Confirm `POSTGRES_PORT` in `.env` matches docker-compose

### Collector Not Collecting
- Check kubeconfig is valid
- Verify Prometheus URL (if provided) â€” leave empty to disable
- Check collector logs: `docker compose -p hvcollector logs hypervisor-collector`

### Fluentd Not Receiving Logs
- Verify port 24224 is reachable from source nodes
- Check Fluentd logs: `docker compose -p hvcollector logs fluentd`
